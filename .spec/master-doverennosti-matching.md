# Мастер-файл доверенностей: авто-слияние и сопоставление с журналом операций 1С

## Контекст
Сейчас сопоставление доверенностей с журналом операций 1С выполняется на основе выгрузки доверенностей как «разового» файла. Файл доверенностей и журнал операций со временем дополняются новыми строками. Требуется накопительный («мастер») файл доверенностей, который хранит результаты сопоставления и позволяет повторно не проверять уже найденные записи.

## Цель
Сделать полностью автоматический сценарий:
1) Пользователь выбирает мастер-файл доверенностей (нарастающий итог).
2) Пользователь выбирает файл доверенностей за текущий/произвольный период (выгрузка).
3) Пользователь выбирает файл выгрузки операций 1С (журнал операций).
Далее макрос:
- добавляет в мастер новые строки доверенностей из периодной выгрузки (без дубликатов);
- сопоставляет строки мастер-файла с журналом операций;
- кэш хранится в мастер-файле в виде колонок результата;
- уже найденные строки не перепроверяются, «не найденные» перепроверяются при каждом запуске.

## Ограничения и правила
- Excel + VBA 7.0, совместимость с Office 2010 (32/64-bit).
- Никаких ActiveCell/Selection/ActiveSheet при наличии альтернатив.
- Обработка ошибок через `On Error GoTo ErrorHandler` в каждой процедуре/функции.
- 2-й и 3-й проходы сопоставления не используются (оставляем только точный 1-й проход).
- Цветовое выделение результатов не используется.
- «Дата проверки» не используется.
- Колонка «Комментарий 1С» обязательна.
- Старые колонки «Процент» и «Детали» не предполагаются (мастер формируется заново).

## Данные и структура мастер-файла
Исходные колонки доверенностей сохраняются как в выгрузке (пример: Дата, Номер, Срок действия, Подотчетное лицо, Организация, Поставщик, Договор, Ответственный, Комментарий).

Добавляемые колонки результатов (ровно 4):
1) **Статус сопоставления** (значения: «НАЙДЕНО», «НЕ НАЙДЕНО»)
2) **Номер операции**
3) **Дата операции**
4) **Комментарий 1С**

## Бизнес-логика сопоставления
- Сопоставление выполняется только для строк, где «Статус сопоставления» пустой или «НЕ НАЙДЕНО».
- Если статус «НАЙДЕНО» — строка пропускается (результат стабилен).
- Поиск совпадения — только точное: номер + дата должны встречаться в комментарии операции 1С.

## Критерии приемки
- Пользователь выбирает 3 файла, после чего мастер автоматически обновляется и сохраняется.
- Новые доверенности добавляются в мастер без дубликатов.
- В мастер добавляются/используются ровно 4 колонки результатов.
- Результаты заполняются без окраски ячеек/строк.
- «НАЙДЕНО» не перепроверяется, «НЕ НАЙДЕНО» и пустые — проверяются.
- В случае найденного совпадения в «Комментарий 1С» записывается полный комментарий операции из журнала.

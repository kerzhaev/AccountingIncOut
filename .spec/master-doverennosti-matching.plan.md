**Примечание:** В процессе реализации было принято решение восстановить полную трехпроходную логику сопоставления вместо упрощенной версии из плана, так как рабочая версия показала лучшие результаты. ## Изменения по функционалу

### 1) Новый «мастер»-сценарий (автоматический)
- [x] Добавить публичную точку входа (макрос) с последовательным выбором 3 файлов:
  1. Мастер-файл доверенностей (нарастающий итог, редактируемый)
  2. Файл доверенностей за период (только чтение)
  3. Файл журнала операций 1С (только чтение)

- [x] После выбора:
  - открыть книги,
  - слить (добавить) новые строки из периодной выгрузки в мастер без дубликатов,
  - выполнить сопоставление с журналом операций,
  - сохранить и закрыть мастер, закрыть файлы-источники.

### 2) Кэш хранится в мастер-файле
- [x] Кэш = заполненные колонки результатов в мастер-файле.

- [x] Правила:
  - статус «НАЙДЕНО» = запись не перепроверяется;
  - статус «НЕ НАЙДЕНО» = запись перепроверяется при каждом запуске;
  - пустой статус = первичная проверка.
  - (дополнительно) статус «ВЕРОЯТНО НАЙДЕНО» также не перепроверяется

### 3) Сопоставление (трехпроходная система - восстановлена рабочая версия)
- [x] Восстановлен полный трехпроходный поиск:
  - 1-й проход: точное совпадение «номер + дата» в комментарии операции (с проверкой слова "НАРЯД").
  - 2-й проход: поиск по поставщику и службе (умный поиск с оценкой уверенности).
  - 3-й проход: универсальные документы (разнарядки, телеграммы, аттестаты, заявки).
- [x] Парсинг номеров и дат:
  - Извлечение номера и даты из комментария доверенности.
  - Проверка наличия слова "НАРЯД" для 1-го прохода.
  - Формат номера: произвольный (цифры, буквы, /, - в любом сочетании).

### 4) Колонки результата в мастер-файле (восстановлены 6 колонок с цветовой маркировкой)
- [x] Восстановлены колонки результатов:
  1. **Трехпроходный поиск** (статус: НАЙДЕНО/ВЕРОЯТНО НАЙДЕНО/ВЕРОЯТНО/НЕ НАЙДЕНО)
  2. **Номер операции**
  3. **Дата операции**
  4. **Процент** (уверенность совпадения)
  5. **Детали** (детали совпадения)
  6. **Комментарий 1С** (полный комментарий операции)

- [x] Требования реализованы:
  - «Комментарий 1С» обязателен и записывается.
  - Цветовая маркировка применена: темно-зеленый (1-й проход), светло-зеленый/желтый (2-й проход), голубой (3-й проход), красный (не найдено).
  - Статусы включают оценку уверенности совпадения.

### 5) Слияние периодной выгрузки в мастер
- [x] Строить индекс по мастер-файлу (в памяти) для быстрой проверки дубликатов.

- `.spec/PROJECT_CONTEXT.md` — обновление контекста проекта после внедрения.

## Изменения по функционалу

### 1) Новый «мастер»-сценарий (автоматический)
Добавить публичную точку входа (макрос) с последовательным выбором 3 файлов:
1. Мастер-файл доверенностей (нарастающий итог, редактируемый)
2. Файл доверенностей за период (только чтение)
3. Файл журнала операций 1С (только чтение)

После выбора:
- открыть книги,
- слить (добавить) новые строки из периодной выгрузки в мастер без дубликатов,
- выполнить сопоставление с журналом операций,
- сохранить и закрыть мастер, закрыть файлы-источники.

### 2) Кэш хранится в мастер-файле
Кэш = заполненные колонки результатов в мастер-файле.

Правила:
- статус «НАЙДЕНО» = запись не перепроверяется;
- статус «НЕ НАЙДЕНО» = запись перепроверяется при каждом запуске;
- пустой статус = первичная проверка.

### 3) Упрощение сопоставления (точность/скорость)
- Оставить только 1-й проход (точное совпадение «номер + дата» в комментарии операции).
- 2-й и 3-й проходы исключить из выполнения (для скорости и исключения ложных срабатываний).
- **ВАЖНО**: Извлечение номеров и дат БЕЗ проверки слова "НАРЯД" или типа документа.
  - Комментарии могут содержать несколько пар "номер от дата".
  - Извлекать ВСЕ пары из комментария и проверять каждую в операциях 1С.
  - Формат номера: произвольный (цифры, буквы, /, - в любом сочетании).
  - Пример: "АС Акт изъятия № 424 от 07.05.2024 г., наряд ВМУ ЮВО № 0307 от 24.05.2024" → извлечь обе пары и проверить каждую.

### 4) Колонки результата в мастер-файле (ровно 4)
Обязательные колонки результатов:
1. **Статус сопоставления**
2. **Номер операции**
3. **Дата операции**
4. **Комментарий 1С**

Требования:
- «Комментарий 1С» обязателен.
- «Дата проверки» не добавлять.
- Цветовую раскраску результатов не применять.
- Старые колонки «Процент» и «Детали» не создаются и не используются (мастер формируется заново).

### 5) Слияние периодной выгрузки в мастер
- Строить индекс по мастер-файлу (в памяти) для быстрой проверки дубликатов.
- Ключ дубликата: нормализованная конкатенация исходных полей строки (A:I) или подмножество, дающее устойчивую уникальность для ваших выгрузок.
- Добавлять только отсутствующие строки, сохраняя исходную структуру столбцов доверенностей.

## Технические изменения в коде (высокоуровнево)
- Заменить `AddThreePassResultColumns` на создание 4-х колонок результата, без заливки/границ; допускается жирный шрифт заголовков.
- Обновить `FindResultColumn` и использование статуса: искать «Статус сопоставления».
- Обновить `WriteThreePassResult`: писать только 4 колонки, без процентов/деталей и без цветов; извлекать и писать полный комментарий операции 1С.
- Обновить `IsAlreadyProcessed`: возвращать True только для статуса «НАЙДЕНО».
- Добавить процедуры «слияния» и «мастер-воркфлоу».
- Обновить тексты статистики/Debug.Print (без упоминания 2/3 проходов и процентов).

### Изменения парсинга номеров и дат (новое требование):
- Добавить структуру `NumberDatePair` для хранения пар номер+дата.
- Создать функцию `ExtractAllNumberDatePairs`: извлечение ВСЕХ пар "номер от дата" из комментария.
  - Искать все вхождения слова "ОТ" (как " ОТ " и "ОТ ").
  - Для каждого "ОТ" извлекать пару: номер (до "ОТ") + дата (после "ОТ").
  - Очищать номера от служебных слов (НАРЯД, ВС, ГСМ, РАВ, БТ, ПС, СС, БПЛА, АС, АКТ и т.д.).
  - Убирать символы "№", "#" из номеров.
  - Поддерживать произвольный формат номера (цифры, буквы, /, -).
- Создать `ExtractNumberBeforeOt`: извлечение номера перед конкретной позицией "ОТ".
- Создать `ExtractDateAfterOt`: извлечение даты после конкретной позиции "ОТ".
- Модифицировать `ParseNaryadForSubstring`:
  - Убрать проверку наличия слова "НАРЯД".
  - Использовать `ExtractAllNumberDatePairs` для извлечения всех пар.
  - Возвращать первую пару как основную (для совместимости).
- Модифицировать `FindBySubstringEnhanced`:
  - Извлекать все пары номер+дата из комментария доверенности.
  - Проверять каждую пару в операциях 1С.
  - Возвращать совпадение при нахождении хотя бы одной пары.

## Проверки после внедрения
- Прогон на тестовых выгрузках:
  - мастер пустой + периодная выгрузка + журнал операций → добавление + сопоставление.
  - повторный запуск с тем же мастер/период/журнал → найденные не трогает, не найденные перепроверяет.
  - добавление новых строк в периодной выгрузке → корректное добавление без дублей + сопоставление только новых/не найденных.
